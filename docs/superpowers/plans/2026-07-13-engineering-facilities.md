# Engineering Facilities Setup Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) for syntax tracking.

**Goal:** Apply developer-experience engineering facilities (lint, format, git hooks, CI, unified version management, TS 7 upgrade, dev docs) to the traceability project.

**Architecture:** The traceability project is a pnpm monorepo with 8 packages across `packages/`, `app/`, `server/`, and `examples/`. Currently it has no linting, no formatting, no git hooks, no CI, and scattered TypeScript/Vitest versions. This plan adds oxlint/oxfmt (Rust tooling), pnpm catalog for version governance, conventional-commits gating via husky, and a basic CI pipeline — all of which are pure DevEx facilities that don't affect application runtime behavior.

**Tech Stack:** pnpm 10.30.3, TypeScript 7.0.2, oxlint 1.61+, oxfmt 0.46+, Husky 9, commitlint 20, lint-staged 16, Vitest (unified), GitHub Actions

## Global Constraints

- No change may affect the application's runtime behavior — all changes are developer-experience only
- pnpm is the exclusive package manager
- Conventionally-committed commit messages (`feat:`, `fix:`, `chore:`, `docs:`, `test:`, `refactor:`, `ci:`)
- oxlint replaces ESLint entirely (no ESLint config)
- oxfmt replaces Prettier entirely (no Prettier config)
- `.husky/` is committed to the repository
- All DevEx config files are committed (no `.gitignore` additions for tool configs)
- pnpm catalog is the single source of truth for shared dependency versions

### Task 6: Add GitHub Actions CI

**Files:**

- Create: `.github/workflows/ci.yml`

**Interfaces:**

- Consumes: pnpm workspace, all packages
- Produces: Automated type-check + test on push to master and PRs

- [ ] **Step 1: Create CI workflow**

**File:** `.github/workflows/ci.yml`

```yaml
name: CI

on:
  push:
    branches: [master, feature/*]
  pull_request:

concurrency:
  group: ${{ github.workflow }}-${{ github.event.number || github.ref }}
  cancel-in-progress: true

jobs:
  validate:
    name: Validate
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4
        with:
          run_install: false
      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: pnpm
      - run: pnpm install --frozen-lockfile
      - name: Type-check all packages
        run: pnpm type-check
      - name: Run all tests
        run: pnpm test
      - name: Lint check
        run: pnpm oxlint --fix-dry-run 2>&1 || true
        # --fix-dry-run reports what would be fixed without writing
        # oxlint exits non-zero when fixes apply; `|| true` keeps CI green
```

````markdown
# AGENTS.md

This file provides guidance to AI agents when working with code in this repository.

## Project Overview

Traceability is a Sentry-based web monitoring + exception-to-fix loop toolkit. It provides:

- **Core** (`packages/core`): rrweb session recording + Sentry error capture integration
- **Protocol** (`packages/protocol`): Shared type definitions between packages

- **React** (`packages/react`): React bindings for traceability components
- **CLI** (`packages/cli`): Command-line interface
- **App** (`app`): Electron desktop application
- **Server** (`server`): Fastify-based server with WebSocket support

## Common Commands

```bash
pnpm dev:app          # Start Electron app in dev mode
pnpm build            # Build all packages
pnpm test             # Run all tests
pnpm type-check       # Type-check all packages
pnpm lint             # Lint all files (oxlint)
pnpm format           # Format all files (oxfmt)
````

## Architecture

| Layer           | Tech                                              |
| --------------- | ------------------------------------------------- |
| Electron App    | Electron 39, React 19, Vite 7, Tailwind CSS 4     |
| Server          | Fastify + WebSocket, Pino logging, better-sqlite3 |
| Core Library    | @rrweb/record, @sentry/browser                    |
| Package Manager | pnpm 10 (workspace + catalog)                     |

## Key Conventions

- **Package Manager**: Strictly `pnpm`. Use `pnpm exec` instead of `npx`
- **Node Linker**: `nodeLinker=hoisted` (flat node_modules via `.npmrc`)
- **Shared Versions**: Managed via pnpm catalog in `pnpm-workspace.yaml`
- **Code Quality**: oxlint (not ESLint) + oxfmt (not Prettier). Config at `oxlint.config.ts` and `oxfmt.config.ts`
- **Git Hooks**: Husky + lint-staged runs `oxlint --fix` and `oxfmt --write` on staged files. commitlint enforces Conventional Commits
- **Testing**: Vitest 4.x. Each package has its own `vitest.config.ts` and test directory (`src/tests/` or `tests/`)
- **TypeScript**: All packages use TS 7.x from catalog. Type-check with `pnpm type-check`
